已经实现的系统

舰船身份与能量网

Fusion Ship Core EnergyGrid

舰船核心注册本机为一艘可被探测的舰船，并提供总能量、装甲/护盾/武器能量分配。运行时 HUD 显示反应堆输出和电容电量。

舰长指挥优先级

Captain MKey

舰长零件已从能量核心拆出。模拟中按 T 切换 AntiAir / AntiShip 优先级，HUD 会显示当前状态。

电容缓冲

Super Capacitor Armor / Shield / Weapon / Universal

电容按能量总线注册容量，能量系统会从对应总线与通用电容中扣除消耗。

雷达轨迹

Radar Panel TrackRegistry

雷达按范围、锥角和 IFF 过滤可追踪对象，生成舰船共享的传感器轨迹和来袭时间估计。

防御指挥

Defense Director FireSolution

防御指挥从传感器轨迹中选择防御目标，当前支持按来袭时间或重威胁优先生成防御解算。

护盾与近防

Shield Projector CIWS Block

护盾消耗护盾能量削弱重型导弹速度并造成少量损伤；CIWS 消耗武器能量，对防御指挥指定的重型导弹造成持续伤害。

重型核导弹目标

Heavy Nuclear Missile ITrackable

重型导弹可手动按 X 发射，或启用 Auto Launch。它注册为重型威胁，可被雷达发现、护盾削弱、CIWS 拦截。

当前限制

范围说明
联网同步第一版主要是主机本地逻辑，尚未做完整多人同步消息。
视觉特效目前主要复用模型和调试线，尚未接入完整枪口、爆炸、护盾场表现。
反舰优先级舰长优先级状态已存在并可切换，但当前防御切片还没有完整反舰武器消费该状态。
伤害模型CIWS/护盾/导弹生命为第一版数值模型，纳米装甲与结构破坏还未接入。

测试建议

防御方放置 Fusion Ship Core、Captain、Super Capacitor、Radar Panel、Defense Director、Shield Projector 和 CIWS Block。攻击方放置 Heavy Nuclear Missile，对准舰船后按 X 发射。观察 HUD 的轨迹数、防御目标、能量和舰长优先级。
