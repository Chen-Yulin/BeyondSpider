# Unified missile launcher, missiles as projectiles

The mod had two separate missile parts: a Heavy Nuclear Missile *block* (a mounted block that flew itself when launched, offensive only) and a Space Interceptor Launcher *block* that spawned small defensive missiles. We collapse both into a single **Missile Launcher** block that fires **missiles as projectiles** — like a cannon round but with a collision box — in three interchangeable sizes.

The launcher picks its size from a build-mode `MMenu` (小型弹 / 中型弹 / 重型弹). The three sizes differ **only in numbers** — maneuverability (turn rate), range, weight (mass), warhead charge, magazine size, reload — never in role: every missile can both intercept incoming threats and strike ships. Which target a given missile guides toward is chosen by **fire control**, not baked into the part. `MissileFireControl.SelectTarget` prioritises the ship's defensive solution (an incoming threat the defence director flagged), then the ship's offensive radar lock, then the nearest hostile in range; a launched missile keeps its assigned target while it lives and re-acquires through the same fire control if the target dies or it was fired blind. Missiles launch along the block's **+forward** axis and steer with the ADR-0008 stack (true PN + closing-speed governor + single rear nozzle, all per-type). All per-type tuning and the cell layouts live as `const`/`static readonly` tables in `MissileLauncherAssets` for hand-tuning; the launcher and projectile logic only read them.

Each launcher is a magazine of cells — **16** (small), **4** (medium), **1** (heavy) — full at simulate start. Firing empties the top cell; an empty magazine refills **one cell at a time** over a per-type reload time, and while any cell is empty the block shows the railgun's twin-circle reload icon. The body has two models: **Close** = bay closed (idle) and **Open** = bay opened for launch. The launch is animated by switching between them: the bay sits closed, pops open when a shot is wanted, fires after a short open lead so the missile emerges from an already-open bay, holds open across a ripple, then closes once nothing is pending. Loaded cells render their missile mesh (a stack: cells `0..loaded-1`) whether the bay is open or closed, so the loadout is always visible and is seen draining and topping back up. Every round mesh — in a cell and in flight — is oriented to point its nose along the block's +forward launch axis, and the cells are laid out in the plane normal to that axis.

A missile is a projectile GameObject carrying a mesh, a `Rigidbody`, a **trigger** `BoxCollider`, and the `MissileProjectile` component (which is both `ITrackable` and `IKineticTarget`). Being a trackable makes it a radar target the enemy's defences can engage; being an `IKineticTarget` with a fixed structural HP makes it shoot-downable exactly like the old missiles. The collision box is a trigger so the missile never physics-pushes a ship, yet raycasting rounds still hit it (`Physics.queriesHitTriggers` is true by default) and it detonates on contact via `OnTriggerEnter` once armed (own ship and friendlies are skipped). Detonation delivers the ADR-0007 warhead blast to nearby armour (charge → linear damage, cube-root radius) plus a direct-hit bonus to any penetrable target it struck. When its structural HP is depleted by point defence the warhead **cooks off** — an intercepted missile still blasts.

The old blocks and their scripts are removed. Every consumer that used to special-case the heavy missile — CIWS, laser, flak turret (all dealing damage to an incoming missile) and the shield projector (decelerating one) — is repointed at `MissileProjectile`, which carries the same `ApplyDamage` / `ApplyShieldDeceleration` / `ThreatMass` surface the heavy missile block did. `ShipState.Launchers` is retyped to `List<MissileLauncherBlock>`.

**Status**: accepted

**Considered Options**

- *Solid collider on the missile* — rejected for a trigger. A solid collider makes the missile physics-collide with its own launching ship on the way out and shove whatever it touches (badly, for the 20-mass heavy round). A trigger has neither problem and is still raycast-hittable for shoot-down and still fires `OnTriggerEnter` for contact detonation. Risk noted: this relies on `Physics.queriesHitTriggers` staying true (its default).
- *Re-run fire control every tick for every in-flight missile* — rejected; that would make a whole salvo u-turn the instant a new threat appeared. Missiles commit to their assigned target and only re-acquire when it is lost, which still honours "fire control decides the target" (at launch and on re-acquire) without the whiplash.
- *Reuse an existing block ID (7 or 11) for the new launcher* — rejected in favour of a fresh ID (12), with 7/11 retired. The mod is unpublished (v0.1, Debug), so save-compatibility does not bind and a clean new ID avoids conflating a fundamentally different part with an old one.
- *Keep the missile a self-flying block* — rejected; a projectile with a collision box is the shape both roles actually want (spawned in quantity, shoot-downable, no per-missile block placement), and it lets one launcher serve attack and interception.
- *Per-cell independent reload timers* — deferred; a single sequential timer refilling a contiguous stack is simpler to render and reason about, and reads the same on the reload icon.

**Consequences**

- Shoot-down depends on `Physics.queriesHitTriggers` (default true). If some other system disables it globally, missiles stop being raycast-hittable and only contact/fuze detonation remains — switch the collider to solid if that ever happens.
- The body-mesh transforms (block XML `<Mesh>` / `<Colliders>`), the per-type mesh scales/rotations, and the **cell layout positions** are placeholders in `MissileLauncherBlock.xml` and `MissileLauncherAssets`; they need in-game tuning against the actual Open/Close/Missile models. The launch axis is `+forward` per design.
- Auto-fire will ripple missiles at a fire-control target as fast as the per-type salvo interval and magazine allow; it does not yet track "this target is already engaged," so it can over-commit rounds. Bounded for now by salvo interval + magazine size.
- The old `BS HeavyMissile *` and `BS Migrated TorpedoLauncher *` resource entries are now unused but left in `Mod.xml` (harmless; TorpedoLauncher may be reused).
- Any new weapon that damages an incoming missile should target the `IKineticTarget` / `MissileProjectile` surface, not a concrete missile block.
